Data Structures in Plain Language · Chapter 6 "Trees"

SCQA · Situation → Conflict → Question → Answer Linear structures cannot model the ubiquitous one-to-many relationships in the real world. Trees far surpass linear lists in complexity — storage, traversal, and search all pose challenges. This chapter uses the binary tree as a bridge: recursive traversal → threading → Huffman coding — three techniques that transform complex tree structures into simple, efficient linear operations, while revealing the principle of lossless data compression.

A tree is a finite set of n (n≥0) nodes, defined recursively: there is exactly one designated root node, and the remaining nodes are partitioned into m (m>0) mutually disjoint subsets, each of which is itself a tree (a subtree). This chapter builds foundational concepts: node/degree/leaf/level/depth/forest; covers three storage representations (parent, child, child-sibling); then dives into binary tree traversal (4 methods), threaded binary trees that repurpose null pointers, and Huffman coding — the theoretical bedrock of data compression.

Recursive Definition
Tree Definition & Core Concepts: From One Tree to a Forest
A tree is a finite set of n (n≥0) nodes, defined recursively. n=0 is the empty tree. A non-empty tree has exactly one root node, with remaining nodes partitioned into m disjoint subtrees. Key concepts: node degree (subtree count), leaf (degree 0), branch node (degree > 0), tree degree (max node degree); child/parent/sibling/ancestor/descendant; level (root at 1) → depth (max level); forest (a set of m disjoint trees). The essential difference from linear lists: one-to-many vs. one-to-one.

Storage Schemes
Three Storage Representations: Parent, Child, Child-Sibling
Parent representation: sequential storage with a parent field — find parent in O(1), find children requires traversal. Extendable with first-child and right-sibling fields. Child representation: sequential header array + per-node child linked list — balances traversal and child finding. Child-sibling representation (binary linked list): each node has firstchild and rightsib pointers — this is the crucial bridge that transforms any tree into a binary tree. Storage design has no silver bullet; choose based on the operations you need.

03
Traversal System
Four Binary Tree Traversals: Turning Trees into Linear Sequences
Each binary tree node has at most two subtrees, ordered left and right. Five basic forms: empty, root-only, left-only, right-only, both children. Special types: skewed trees (left/right), full binary tree (all levels full), complete binary tree (level-order numbering has no gaps). Five key properties: layer i has ≤2ⁱ⁻¹ nodes; depth k has ≤2ᵏ-1 nodes; n₀=n₂+1; depth ⌊log₂n⌋+1; parent-child indices 2i & 2i+1. Four traversals linearize the tree: Pre-order (root→left→right), In-order (left→root→right), Post-order (left→right→root), Level-order (top-down, left to right). Pre+In or Post+In uniquely determine a binary tree; Pre+Post alone cannot.

Space Optimization
Threaded Binary Trees: Turning Null Pointers into GPS Navigation
An n-node binary linked list has 2n pointer fields but only n-1 branches — leaving n+1 null pointers wasted. Threading repurposes these null pointers to store the predecessor and successor of each node under a specific traversal order — like a GPS that tells you where to go next. Add ltag/rtag flags to distinguish "points to child" (0) from "points to thread" (1). A threaded binary tree is effectively a doubly linked list, enabling O(n) traversal without ever needing to re-traverse to find predecessors/successors. Adding a head node allows traversal from the first node forward to the last, or from the last backward to the first.

Compression
Huffman Tree & Coding: The Mathematical Beauty of Data Compression
A Huffman tree is the binary tree with the minimum weighted path length (WPL). Construction: repeatedly merge the two smallest-weight nodes into a new parent whose weight is their sum. Huffman coding: build a Huffman tree using character frequencies as weights; assign 0 to left branches, 1 to right branches; the root-to-leaf path is the character's code. Key property: no character's code is a prefix of another's (prefix code) — guaranteeing unambiguous decoding. Example with frequencies A27/B8/C15/D15/E30/F5: Huffman coding compresses a 30-bit binary string to 25 bits — ~17% space saved. This is the theoretical foundation of lossless compression underlying ZIP, JPEG, and modern codecs.

①Sort weights ②Merge two smallest ③Repeat until one tree remains → WPL=40×1+30×2+15×3+10×4+5×4=205 (optimal)

🔑 Key Takeaway
Trees transcend the one-to-one limit of linear structures through recursive definition. The binary tree is the chapter's hub: four traversals linearize trees for computer processing; threading repurposes n+1 null pointers for O(n) backtrack-free traversal; the child-sibling representation unifies arbitrary trees as binary trees. The Huffman tree minimizes WPL; Huffman coding achieves lossless compression through prefix codes — Huffman's 1952 discovery still underpins ZIP, JPEG, and modern codecs today. The equivalence transformation among trees, forests, and binary trees proves that complex problems have simple solutions.
