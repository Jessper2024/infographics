从Excel到Pandas的思维转换

第3章标志着从\"学工具\"到\"用工具\"的转折点。从此章起，每一章都围绕数据分析流程中某个具体操作，对比Excel和Python的实现方式。

  • 三大模块约定：Pandas → pd，NumPy → np，matplotlib.pyplot → plt。全书统一使用别名，一个程序只需导入一次
  • Excel思维 → Pandas思维：Excel的一列 ≈ Series；Excel的一张工作表 ≈ DataFrame。行索引 + 列索引确定唯一值，与Excel的行号列号概念完全对应
  • 关键差异：Excel行/列索引计入行列数（含表头），Python的shape方法不计索引
2
Series：一维带标签数组

Series由一组数据（值）和一组相关数据标签（索引）组成，类似于带行号的单列。

  • 创建Series — pd.Series()：
    ① 传入列表 → 默认索引从0开始
    ② 传入列表 + index参数 → 自定义数据标签
    ③ 传入字典 → key为数据标签，value为数据值
  • 获取索引：series.index 方法
  • 获取值：series.values 方法
  • Excel类比：Series就相当于Excel中的某一列（不含列名），左侧的索引对应行号
3
DataFrame：二维表格型结构

DataFrame由一组数据与一对索引（行索引和列索引）组成，是最核心的表格型数据结构。之所以叫\"表格型\"，是因为它的数据形式与Excel的数据存储形式高度相似。

  • 创建DataFrame — pd.DataFrame()：
    ① 传入单一列表 → 单列数据，默认行列索引
    ② 传入嵌套列表 → 多列数据（每层嵌套为一列）
    ③ 传入嵌套列表 + columns参数 → 自定义列索引
    ④ 传入嵌套列表 + index参数 → 自定义行索引
    ⑤ 传入字典 → key为列索引，value为各列数据
  • Excel类比：DataFrame ≈ 一张完整的Excel工作表，行索引=行号，列索引=列名（表头）
4
DataFrame的行列索引与值获取

掌握DataFrame的行列索引获取是后续数据选择操作的前提：

  • 获取列索引：df.columns — 返回所有列名。类似查看Excel的表头行
  • 获取行索引：df.index — 返回所有行标签。类似查看Excel的行号列
  • 获取值：df.values — 以数组形式返回DataFrame中的纯数据（不含索引）。这是第6章\"数据选择\"的基础

理解行索引和列索引是后续所有数据操作的基础——无论是数据选择（第6章）、数值替换（第7章）、数据运算（第8章），还是多表拼接（第11章），本质上都是在行列索引的坐标系中定位和操作数据。

5
模块导入与全书编码约定

三个核心模块的导入是全书所有Python代码的前置条件：

  • import pandas as pd — Pandas是全书最核心模块，几乎每章必用
  • import numpy as np — NumPy是数值计算基础，Pandas基于NumPy构建
  • import matplotlib.pyplot as plt — matplotlib用于数据可视化（第13章）

代码示例中，pd.Series()、pd.DataFrame()、pd.read_excel() 等写法中的\"pd\"就是Pandas的别名。使用as起别名是为了书写方便——这也是Python社区的通用惯例。

💡 核心结论

Series和DataFrame是Pandas的两块基石。如果你理解Excel的行列概念，就很容易理解Pandas数据结构——Series = 一列数据 + 行索引，DataFrame = 多列数据 + 行索引 + 列索引。本章虽然短，但它是后续所有操作（导入、清洗、选择、运算、分组、拼接、可视化）的\"语法基础\"。记住：pd/np/plt 三个别名一旦导入，全书代码都可复用——这就是Python模块化的力量。
