对比Excel，轻松学习Python报表自动化 · 第3章「用Python对报表进行基本操作」

SCQA · 情境 → 冲突 → 问题 → 回答 日常工作需要频繁地打开、创建、写入Excel文件，并对Sheet进行管理。手工操作虽简单，但当成百上千个文件需要处理时就变得不可能。如何用Python替代这些基础的Excel操作？本章对比讲解了load_workbook()/read_excel()两种读取方式，Workbook()创建工作簿，append()写入数据，以及Sheet的创建、重命名、复制与删除——每个操作都配有Excel界面对照和Python代码，让读者直观理解映射关系。

本章从最基础的Excel文件操作入手，建立Excel界面操作与Python代码之间的映射。首先对比了openpyxl的load_workbook()和Pandas的read_excel()两种读取方式——前者一次性加载所有Sheet，后者仅读取指定Sheet但结果显示更直观。然后介绍了如何创建新工作簿并插入数据：支持单元格赋值（ws['A1']=value）、整行赋值（append()）、多行赋值（for循环+append()）。还解决了一个常见痛点：Pandas与openpyxl之间的数据格式转换，通过dataframe_to_rows()函数实现两个库的无缝衔接。最后覆盖了Sheet管理：create_sheet()新建、title属性重命名与标签颜色、copy_worksheet()复制、以及删除操作。

01
文件读取
打开工作簿：load_workbook() vs read_excel()
openpyxl的load_workbook()一次性将工作簿所有Sheet导入Python，但不会实时显示数据。Pandas的read_excel()可将数据直接展示在界面上，但一次只能读取一个Sheet（默认第一个，可用sheet_name参数指定）。核心区别：load_workbook适用于需要操作多个Sheet的场景，read_excel适合快速查看和数据分析。

文件创建
创建工作簿与三种数据写入方式
Workbook()声明新工作簿，wb.save(path)保存到本地。数据写入有三种粒度：单格赋值（ws['A1']=1 或 ws.cell(row=1,column=1).value=1）、整行赋值（ws.append([1,2,3])一次性插入一行）、多行赋值（for循环+append()批量插入），灵活适配不同数据规模。

03
库间桥接
Pandas与openpyxl之间的数据转换
实际工作中常需混用Pandas和openpyxl：先用Pandas处理数据，再用openpyxl设置格式。dataframe_to_rows()是连接两个库的桥梁：将DataFrame转化为逐行数据，参数index控制是否保留索引列，header控制是否保留列名。然后用openpyxl的append()将转换后的数据逐行写入工作簿。
# Pandas读取 → openpyxl写入
from openpyxl.utils.dataframe import dataframe_to_rows

df = pd.read_excel("dataset.xlsx")
wb = Workbook()
ws = wb.active

for row in dataframe_to_rows(df, index=False, header=True):
ws.append(row)

wb.save("output.xlsx")

05
实战应用
从零创建一张完整报表
将这些基础操作串联起来，就能从零创建一张完整报表：先创建工作簿，用append()写入表头和数据行，再根据需要新建多个Sheet并命名，最后保存。这看似简单，但当你需要每天生成一份日报时，这段代码的价值就体现出来了——一次编写，永久复用。

🔑 核心结论
Python操作Excel文件的基础API并不复杂——load_workbook()打开、Workbook()创建、append()写入、save()保存。真正的价值在于将这些简单操作组合成自动化流程，让一个脚本替代成百上千次手工点选。掌握dataframe_to_rows()，就能在Pandas和openpyxl之间自由切换。
