Compare with Excel, Easily Learn Python Report Automation · Ch.3 "Basic Excel Operations with Python"

SCQA · Situation → Complication → Question → Answer Daily work involves frequently opening, creating, writing Excel files, and managing sheets. Manual operations are simple but become impossible with hundreds of files. How can Python replace these basic Excel operations? This chapter compares load_workbook() vs read_excel() for reading, Workbook() for creation, append() for writing data, and sheet management—creation, renaming, copying, and deletion—each with Excel UI comparison and Python code for intuitive mapping.

This chapter starts with the most basic Excel file operations, building the mapping between Excel UI actions and Python code. It first compares two reading methods: openpyxl's load_workbook() loads all sheets at once but doesn't display data, while Pandas' read_excel() shows data immediately but reads one sheet at a time. It then covers creating workbooks and writing data in three granularities: cell assignment (ws['A1']=value), row insertion (append()), and multi-row insertion (for loop + append()). It also solves a common pain point: data format conversion between Pandas and openpyxl via dataframe_to_rows(). Finally it covers sheet management: create_sheet(), renaming, tab coloring, copy_worksheet(), and deletion.

File Reading
Opening Workbooks: load_workbook() vs read_excel()
load_workbook() from openpyxl imports all sheets at once but doesn't render data. read_excel() from Pandas displays data immediately but reads only one sheet per call (defaults to first; use sheet_name parameter for others). Key distinction: load_workbook suits multi-sheet operations; read_excel suits quick data viewing and analysis.

Creating Workbooks & Three Data Writing Methods
Workbook() declares a new workbook; wb.save(path) saves locally. Three writing granularities: single cell (ws['A1']=1 or ws.cell(row,col).value=1), row insertion (ws.append([1,2,3])), and multi-row insertion (for loop + append()). These flexibly adapt to different data scales.

Library Bridge
Data Conversion Between Pandas & openpyxl
Real projects often mix Pandas and openpyxl: process data with Pandas, format with openpyxl. dataframe_to_rows() is the bridge: it converts a DataFrame into row-by-row data. Parameters: index controls whether the index column is kept; header controls column names. Then use openpyxl's append() to write converted rows into the workbook.
# Pandas read → openpyxl write from openpyxl.utils.dataframe import dataframe_to_rows df = pd.read_excel("dataset.xlsx") wb = Workbook() ws = wb.active for row in dataframe_to_rows(df, index=False, header=True): ws.append(row) wb.save("output.xlsx")

Create, Rename, Copy & Delete Sheets
create_sheet() adds a new sheet (specify title and index position). copy_worksheet(source) duplicates an existing sheet. ws.title = 'NewName' renames; ws.sheet_properties.tabColor sets tab color. wb.remove(ws) or del wb[name] deletes a sheet. wb.sheetnames returns a list of all sheet names in the workbook.

🔑 Key Takeaway
The basic Python API for Excel file operations isn't complicated: load_workbook() to open, Workbook() to create, append() to write, save() to persist. The real value lies in combining these simple operations into automated workflows, replacing hundreds of manual clicks with a single script. Master dataframe_to_rows() to seamlessly switch between Pandas and openpyxl.
