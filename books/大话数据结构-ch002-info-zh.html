算法定义
算法的五大特性：什么才算"算法"？
算法是解决特定问题求解步骤的描述，在计算机中表现为指令的有限序列，每条指令表示一个或多个操作。算法必须具备五大特性：（1）输入——有零个或多个输入；（2）输出——至少有一个输出；（3）有穷性——执行有限步骤后自动结束，不会无限循环；（4）确定性——每一步都有确定含义，不会出现二义性；（5）可行性——每一步都能通过有限次执行完成。同时满足这五个条件的求解步骤才是合格的算法。

02
效率对比
高斯求和：两种算法 × 天壤之别
计算1+2+…+100：第一种算法用循环累加（执行100次加法，O(n)）；第二种用高斯公式n(n+1)/2（一次乘法+一次除法+一次加法，O(1)）。这不仅是算法优劣的对比，更是思维方式的分水岭。高斯小时候能想到公式法，不是因为他知道"时间复杂度"这个概念，而是因为他具备抽象思维——从具体问题中发现模式、归纳规律。算法设计的本质，就是用更聪明的方式让计算机少做事。
🐢

循环累加法 O(n)

int sum=0; for(i=1;i<=100;i++) sum+=i;
执行100次加法 → 时间复杂度 O(n)

🚀

高斯公式法 O(1)

sum = n*(n+1)/2 = 100*101/2 = 5050
1次乘法+1次加法+1次除法 → O(1)

03
效率度量
算法效率的度量：从事后统计到事前分析
事后统计法：写完程序后运行计时——严重依赖硬件、数据规模、编译优化，不可靠。事前分析估算法：在程序编写前，依据统计方法对算法进行估算。核心洞察：一个程序的运行时间依赖于算法的好坏和问题的输入规模，而最终落脚点在基本操作的执行次数上。因此，我们忽略所有常数和低阶项，只关注输入规模n趋于无穷时增长最快的项——这就是大O阶的核心思想。
"测定运行时间最可靠的办法，就是计算对运行时间有消耗的基本操作的执行次数。运行时间与这个计数成正比。" — 程杰
04
大O阶推导
时间复杂度：推导大O阶的三步法
推导大O阶：①用常数1取代运行时间中的所有加法常数；②只保留最高阶项；③去掉最高阶项的系数。例如：3n²+2n+1 → O(n²)。常数阶（无循环）= O(1)，线性阶（一层循环）= O(n)，对数阶（每次乘2）= O(log n)，平方阶（两层嵌套循环）= O(n²)。理解这一点：大O阶不关心具体数字，只关心当n→∞时的增长速度。
📐 三步法示例：
原始执行次数：f(n) = 3n² + 100n + 2000
① 用1替换常数：3n² + 100n + 1
② 保留最高阶：3n²
③ 去掉系数：n²
→ O(n²)
05
复杂度全景
常见时间复杂度的全排序与场景
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)。O(1)是哈希查找的理想，O(log n)见于二分查找，O(n)是线性遍历，O(n log n)是归并/快速排序，O(n²)是冒泡/选择排序，O(n³)是三重循环，O(2ⁿ)是子集枚举，O(n!)是全排列。此外还有算法的空间复杂度——计算算法所需存储空间的量度。时间换空间还是空间换时间，取决于具体场景的资源约束。

🔑 核心结论
算法的质量决定了程序的效率。五大特性和四大设计要求是检验算法的标准。算法效率分析的核心是大O时间复杂度——通过推导大O阶，忽略常数和低阶项，只关注增长趋势。常见时间复杂度从O(1)到O(n!)共有8个层级，不同复杂度的算法在面对大规模数据时差距惊人。好的算法=用更聪明的方式让计算机少做事。学会分析时间复杂度，是成为优秀程序员的必修课。
