Data Structures in Plain Language · Chapter 5 "Strings"

SCQA · Situation → Conflict → Question → Answer Computers evolved from scientific computation to non-numeric processing, demanding character manipulation. Yet naive pattern matching degrades catastrophically under partial-match scenarios. How do we match without backtracking the main string? This chapter introduces the KMP algorithm, which leverages prefix-suffix similarity to avoid backtracking, and the nextval array for further optimization — reducing pattern-matching complexity from O((n-m+1)·m) to O(n+m).

A string is a finite sequence of zero or more characters — a character-oriented extension of the linear list. Unlike linear lists that focus on single elements, strings center on substring search, replacement, and matching. The core breakthrough is the KMP pattern-matching algorithm: by preprocessing the pattern string into a next array, the main string index i never backtracks, reducing worst-case complexity from O((n-m+1)·m) to O(n+m). The improved nextval array further eliminates redundant comparisons caused by repeated characters.

Fundamentals
String Definition & Comparison: Mathematical Abstraction of Character Sequences
A string is a finite sequence of zero or more characters, denoted s="a₁a₂…aₙ" with length n. An empty string has length zero ("" or Φ); a space string is a non-empty string of spaces. A substring is any contiguous sub-sequence of characters; its position is the index of its first character in the main string. Strings are compared lexicographically via character encoding: s=t ⇔ n=m and every aᵢ=bᵢ. When unequal, compare character by character until the first difference — if the shorter string is a prefix of the longer, the shorter is smaller.

Storage Schemes
String Storage Structures: Fixed-Length Arrays vs. Linked Allocation
Sequential storage uses a fixed-length array, storing the length at index 0 or using '\0' as a terminator. The problem is that operations like Concat, StrInsert, and Replace may exceed MaxSize, causing truncation or overflow. Dynamic allocation (heap via malloc/free) mitigates this. Linked storage lets each node hold one or multiple characters, padding unfilled nodes with '#'. Node size significantly affects processing efficiency. Overall, linked storage is less flexible and underperforms sequential storage except for concatenation.

Brute Force
Naive Pattern Matching: The Cost of Double Loops
The naive algorithm treats each position i in the main string S as a starting point, comparing against pattern T character by character. Best case: match at first position → O(1). Average case: first character mismatch → O(n+m). Worst case: every mismatch is discovered at T's last character (e.g., S="000…001" vs. T="0000000001") → O((n-m+1)·m). This inefficiency is particularly devastating when processing binary 0/1 strings in computing.

Core Breakthrough
KMP Pattern Matching: A Good Horse Never Looks Back
The KMP algorithm's core idea: the main-string index i never backtracks; only j changes. When T="abcdex" (no repeated characters), upon mismatch j can reset to 1 immediately. When T="abcabx" (prefix "ab" = suffix "ab"), j drops from 6 to 3. next[j] records the longest equal prefix-suffix length before position j in T: one matching character → k=2; n matching → k=n+1. The algorithm has two phases: ① build next array O(m), ② pattern matching O(n): total O(n+m). KMP excels when many partial matches occur.

05
Refinement
nextval Improvement: Eliminating Redundant Backtracking
KMP still has a flaw: when T contains consecutive identical characters (like "aaaaax"), next backtracking produces redundant comparisons. Example: S="aaaabcde" matching T="aaaaax" — after mismatch at i=5,j=5, j steps through 4→3→2→1, each comparing "b" vs. "a" redundantly. The nextval fix: during next computation, if T[i]==T[next[i]], then nextval[i]=nextval[next[i]], skipping unnecessary backtracking. T="aaaaaaaab" → next=012345678 → nextval=000000008, dramatically reducing redundant steps.

🔑 Key Takeaway
Strings extend linear lists into the character domain, with operations centered on substring search and matching. Naive pattern matching's worst-case O((n-m+1)·m) is disastrous for binary data. KMP achieves O(n+m) by preprocessing the next array and never backtracking the main string. The nextval improvement eliminates redundant comparisons for repeated-character patterns. The palindrome poem "Xuanji Tu" uses 841 characters to form 7,958 poems — string reversal and matching are both an art of algorithms and a cipher of culture.
