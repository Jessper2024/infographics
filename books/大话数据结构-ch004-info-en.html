Stack Definition & Implementation
Stack Definition and Sequential Storage: The Essence of LIFO
Stack: a linear list restricted to insertion and deletion only at the tail end. The operable end is called the top; the other end is the bottom. LIFO (Last In First Out) structure. Analogy: a revolver cylinder — the last bullet pushed in is the first fired. Sequential stack: implemented with an array, the top variable indicates the stack top position. Empty stack: top=-1; Full stack: top=MAXSIZE-1. Push: S->top++; S->data[S->top]=e. Pop: *e=S->data[S->top]; S->top--. Two operations, both O(1).

Shared Space & Linked Stack
Two Stacks Sharing Space and Linked Stacks: More Flexible Options
Two stacks sharing space: one array stores two stacks, with bottoms at opposite ends (index 0 and n-1), growing toward each other. Full condition: top1+1==top2. Like two roommates sharing a two-bedroom apartment — sharing common space is more efficient than each renting a separate one-bedroom. Linked stack: a linked list implementation with the stack top at the list head. Push = head insertion; Pop = remove head node. Biggest advantage of linked stacks: no pre-allocation, no overflow (unless memory exhausted). Sequential stacks suit stable element counts seeking speed; linked stacks suit varying counts seeking space efficiency.
🏠 Analogy: Two Stacks Sharing Space
Two college roommates share a two-bedroom apartment — each has their own bedroom, shares the living room and kitchen — cheaper and more space-efficient than two separate one-bedrooms. Similarly, one array holds two stacks, growing from ends toward center for maximum utilization.

Stack Applications
Stack Applications: From Recursion to Expression Evaluation
Recursion: a function that calls itself directly or indirectly through a chain of calls. Recursive execution is essentially a stack push-pop process: each function call pushes parameters and local variables onto a stack frame; returning pops them. Using the Fibonacci sequence F(n)=F(n-1)+F(n-2) as an example — although recursive code is elegant, heavy repeated computation makes it less efficient than iteration. Four-arithmetic expression evaluation: infix → postfix (Reverse Polish Notation), then evaluate with a stack. Postfix requires no parentheses: push numbers when encountered, pop two operands and compute when an operator appears — this is the core principle behind compilers and calculators.

04
Queue & Circular Queue
Queue Definition and Circular Queue: The Art of FIFO
Queue: a linear list allowing insertion only at one end and deletion only at the other. FIFO (First In First Out). Analogy: buying tickets in line — the first to arrive is the first served. Sequential queue: array implementation, front points to the head, rear points to the next position after the tail. But sequential queues suffer "false overflow": rear reaches the array end while front has available space, preventing further enqueuing. Circular queue: connect array ends to form a ring. rear=(rear+1)%MAXSIZE enables circular movement. Empty: front==rear; Full: (rear+1)%MAXSIZE==front (reserve one slot to distinguish empty from full).
⚠️ False Overflow Problem:
In a sequential queue, when rear reaches the array's end, no more elements can be enqueued even if front has freed space — this is "false overflow." A circular queue transforms the array into a ring via modulo: (rear+1) % QueueSize — rear wraps back to 0, reusing space.
05
Linked Queue
Linked Storage for Queues: Implementing Linked Queues
Linked queue: a queue implemented with a linked list, using front to point to the head node and rear to the tail node. Enqueue: insert at tail (rear->next=new_node; rear=new_node). Dequeue: remove head (front=front->next). Empty check: front==rear (both point to the header node). Linked queues have no "false overflow" problem and need no pre-allocation — ideal for scenarios with highly variable element counts. When the maximum size is known, prefer circular queues; when size is unpredictable, use linked queues.

🔑 Key Takeaway
Stacks (LIFO) and queues (FIFO) are two restricted linear lists — limiting operation positions is not a weakness but a source of precise control. Stack push/pop are both O(1); two stacks sharing space improves array utilization; linked stacks have no capacity limits. Stack applications include recursion and expression evaluation (postfix notation). Queues solve false overflow with circular queues, and linked queues dynamically allocate space. Core insight: restriction is not weakening — by precisely limiting operation positions, stacks and queues each become powerful tools for specific problems.
