Importing .xlsx Files: From Double-Click to read_excel()

In Excel: double-click to open. In Python: call read_excel(). Behind this seemingly simple step lies rich parameter control:

  • Basic Import: specify file path (Windows paths: use r prefix for escaping or replace \ with /)
  • sheet_name: specify Sheet by name or order number (0-indexed). Defaults to first Sheet—solves multi-sheet workbook scenarios
  • index_col: use a specific column (0-indexed) as the row index. e.g., use the "ID" column instead of default 0,1,2...
  • header: specify which row serves as column index. Default header=0 (first row as headers); can set to other rows or None
  • usecols: import only needed columns (single value or list). Dramatically improves efficiency when source tables have too many columns
2
Importing .csv Files: Separators, Encoding & Engines

read_csv() is the most frequently used import function—.csv complexity far exceeds .xlsx:

  • sep (Separator): defaults to comma. Other common separators: space, tab (\t). Mismatched separator = data fails to split correctly, all content squeezed into one cell
  • nrows (Row Count): for multi-hundred-megabyte files, read only the first N rows for preview—no need to import everything
  • encoding (Character Encoding): UTF-8 and gbk are the two most common encodings. Same file saved with different encodings requires different parameters—CSV UTF-8 uses default or utf-8-sig; CSV (comma-delimited) uses gbk
  • engine (Parser Engine): when file paths contain Chinese characters, switch from default C engine to Python engine. Pair with encoding adjustment to utf-8-sig or gbk
  • Row/Column Indexing & Column Selection: same as .xlsx import (index_col, header, usecols)
3
Importing .txt & SQL Files: Universal Functions & Database Connections

read_table() is the universal function for importing delimiter-separated files into DataFrame; read_sql() bridges databases and Python:

  • .txt File Import (read_table):
    Excel: Data → Get External Data → From Text → Choose delimiter → Finish (multi-step wizard)
    Python: read_table(path, sep='delimiter')—even comma-delimited requires explicit sep, unlike read_csv()
    read_table handles both .txt and .csv; parameters align with read_csv()
  • SQL File Import (read_sql):
    Excel: Data → From Other Sources → Choose database type → Configure connection
    Python two-step: ① pymysql.connect() establishes connection (pip install pymysql first) ② pd.read_sql(SQL query, con=connection_object). Supports index_col and columns parameters
4
Creating New Data & The 4-Step Data Familiarization Routine

Beyond importing external data, sometimes you need to create DataFrames from scratch. Don't rush into analysis—use the 4-step routine to thoroughly understand your data first:

  • Create New DataFrame: pd.DataFrame() (covered in Ch.3). Pass lists/dictionaries to create. Used for testing, demos, or small-scale data construction
  • ① head() — Preview First N Rows:
    Excel: open file, full display, scroll for large tables
    Python: df.head(N), defaults to 5 rows. Quickly see what each column's data looks like
  • ② shape — Get Table Dimensions:
    Excel: select a column/row, check count in the bottom-right corner
    Python: df.shape → returns (rows, cols) tuple. Note: Python excludes index counts; Excel includes them
  • ③ info() — Check Data Types:
    Excel: select each column individually → check Number group in the ribbon
    Python: df.info() → outputs all columns' data types, non-null counts, and memory usage in one shot
  • ④ describe() — Numerical Distribution Summary:
    Excel: select column → bottom-right shows only average/count/sum
    Python: df.describe() → retrieves count/mean/std/min/25%/50%/75%/max for all numeric columns at once
5
Excel vs Python: Complete Import Operation Comparison

Summary of Excel and Python import methods for all four formats:

  • .xlsx: Excel: double-click | Python: read_excel() + sheet_name/index_col/header/usecols
  • .csv: Excel: double-click | Python: read_csv() + sep/encoding/engine/nrows/index_col/header/usecols
  • .txt: Excel: multi-step wizard | Python: read_table() + sep (universal delimiter function)
  • SQL: Excel: Data Connection Wizard | Python: pymysql.connect() + read_sql() (two-step)

Core Pattern: Excel operations = point-and-click/wizards (WYSIWYG, good for ad-hoc tasks); Python code = repeatable, automatable, handles large files (ideal for production). Both achieve the same goal—the choice depends on context.

💡 Key Takeaway

Data import is the "first station" on the data analysis assembly line. Mastering read_excel/read_csv/read_table/read_sql gives you the keys to unlock data from any source. It's not about memorizing every parameter—it's about understanding each parameter's "business scenario": usecols solves "too many columns," nrows solves "file too large," encoding solves Chinese character garbling, sep solves delimiter mismatches. After importing, always run the head→shape→info→describe routine—this is the shortest path to judging whether the data you've acquired can meet your analysis needs.
