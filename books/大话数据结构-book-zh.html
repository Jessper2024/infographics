第1章 · 数据结构绪论

本章是全书的总纲。作者通过小菜在工作中用错数据结构的真实故事，引出"数据结构很重要"的核心命题。本章建立了完整的概念体系：从数据→数据元素→数据项→数据对象→数据结构，再到四大逻辑结构（集合、线性、树形、图形）与两大物理存储结构（顺序存储、链式存储），最后引入抽象数据类型（ADT），将"是什么"与"能做什么"统一起来。本章还追溯了1968年Knuth教授开创数据结构课程体系的历史。

概念体系
从数据到数据结构：五层概念塔
数据（Data）：描述客观事物的符号，能输入计算机处理的符号集合。数据元素（Data Element）：组成数据的基本单位，通常作为整体处理。数据项（Data Item）：数据元素的不可分割的最小单位。数据对象（Data Object）：性质相同的数据元素的集合，是数据的子集。数据结构（Data Structure）：相互之间存在一种或多种特定关系的数据元素的集合。这个五层概念体系是理解所有后续章节的基础。

03
逻辑结构
四大逻辑结构：数据元素的组织方式
逻辑结构是数据元素之间的相互关系，分为四类：集合结构（元素同属一个集合，无其他关系）、线性结构（一对一关系）、树形结构（一对多的层次关系）、图形结构（多对多的任意关系）。逻辑结构面向问题，是抽象的数据组织形式。例如：排队中的小朋友是线性结构，公司组织架构是树形结构，城市交通网络是图形结构。这四种结构覆盖了几乎所有现实世界的建模需求。

04
物理存储
两大物理结构：顺序存储与链式存储
物理结构（存储结构）是数据结构在计算机内存中的实际存储方式。顺序存储结构：把数据元素存放在地址连续的存储单元里，数据间的逻辑关系和物理关系一致——如同排队时每个人紧挨着站在一起。链式存储结构：把数据元素存放在任意的存储单元里，通过指针来连接——如同医院候诊时每人手里拿一个号码牌，通过号码关联。顺序存储适合大量读取，链式存储适合频繁插入删除。

🔗

链式存储结构

05
抽象数据类型
抽象数据类型(ADT)：从"是什么"到"能做什么"
抽象数据类型（Abstract Data Type, ADT）：一个数学模型及定义在该模型上的一组操作。ADT不关心具体实现，只关心"是什么"和"能做什么"。例如：整型的ADT包含加减乘除等操作，而不关心底层如何存储。线性表的ADT包含初始化、查找、插入、删除等操作。ADT是连接"数据结构概念"与"具体编程实现"的桥梁——先定义数据能做什么，再实现怎么做。这体现了面向对象思想的核心：封装与抽象。
🧠 核心理解：ADT = 数据对象 + 数据关系 + 基本操作。"抽象"的意义在于隐藏实现复杂性，只暴露可操作的接口。"数据类型"的含义是它和int、float一样，提供一组定义好的操作。
🔑 核心结论
数据结构是相互之间存在一种或多种特定关系的数据元素的集合。四大逻辑结构（集合、线性、树形、图形）加上两种物理存储方式（顺序、链式），构成了一切数据结构知识的基础。抽象数据类型（ADT）将"数据的定义"与"数据的操作"封装在一起，是面向对象思想的基石。学数据结构，不是为了考试，而是为了在工作中"用正确的方式解决已经解决的问题"。

第2章 · 算法

本章从高斯求和的故事出发，揭示了一个核心事实：解决同一个问题，不同算法的效率可以相差数十万倍。作者系统讲解了算法的五大特性（输入、输出、有穷性、确定性、可行性）和四大设计要求（正确性、可读性、健壮性、高效性），然后重点展开算法效率的度量方法——从事后统计的局限到事前分析的优越性，推导出大O阶表示法。最后，用函数的渐近增长概念串起常见时间复杂度的排序：O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)。

01
算法定义
算法的五大特性：什么才算"算法"？
算法是解决特定问题求解步骤的描述，在计算机中表现为指令的有限序列，每条指令表示一个或多个操作。算法必须具备五大特性：（1）输入——有零个或多个输入；（2）输出——至少有一个输出；（3）有穷性——执行有限步骤后自动结束，不会无限循环；（4）确定性——每一步都有确定含义，不会出现二义性；（5）可行性——每一步都能通过有限次执行完成。同时满足这五个条件的求解步骤才是合格的算法。

效率对比
高斯求和：两种算法 × 天壤之别
计算1+2+…+100：第一种算法用循环累加（执行100次加法，O(n)）；第二种用高斯公式n(n+1)/2（一次乘法+一次除法+一次加法，O(1)）。这不仅是算法优劣的对比，更是思维方式的分水岭。高斯小时候能想到公式法，不是因为他知道"时间复杂度"这个概念，而是因为他具备抽象思维——从具体问题中发现模式、归纳规律。算法设计的本质，就是用更聪明的方式让计算机少做事。
🐢

执行100次加法 → 时间复杂度 O(n)

🚀

高斯公式法 O(1)

sum = n*(n+1)/2 = 100*101/2 = 5050
1次乘法+1次加法+1次除法 → O(1)

03
效率度量
算法效率的度量：从事后统计到事前分析
事后统计法：写完程序后运行计时——严重依赖硬件、数据规模、编译优化，不可靠。事前分析估算法：在程序编写前，依据统计方法对算法进行估算。核心洞察：一个程序的运行时间依赖于算法的好坏和问题的输入规模，而最终落脚点在基本操作的执行次数上。因此，我们忽略所有常数和低阶项，只关注输入规模n趋于无穷时增长最快的项——这就是大O阶的核心思想。
"测定运行时间最可靠的办法，就是计算对运行时间有消耗的基本操作的执行次数。运行时间与这个计数成正比。" — 程杰
04
大O阶推导
时间复杂度：推导大O阶的三步法
推导大O阶：①用常数1取代运行时间中的所有加法常数；②只保留最高阶项；③去掉最高阶项的系数。例如：3n²+2n+1 → O(n²)。常数阶（无循环）= O(1)，线性阶（一层循环）= O(n)，对数阶（每次乘2）= O(log n)，平方阶（两层嵌套循环）= O(n²)。理解这一点：大O阶不关心具体数字，只关心当n→∞时的增长速度。
📐 三步法示例：
原始执行次数：f(n) = 3n² + 100n + 2000
① 用1替换常数：3n² + 100n + 1
② 保留最高阶：3n²
③ 去掉系数：n²
→ O(n²)
05
复杂度全景
常见时间复杂度的全排序与场景
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)。O(1)是哈希查找的理想，O(log n)见于二分查找，O(n)是线性遍历，O(n log n)是归并/快速排序，O(n²)是冒泡/选择排序，O(n³)是三重循环，O(2ⁿ)是子集枚举，O(n!)是全排列。此外还有算法的空间复杂度——计算算法所需存储空间的量度。时间换空间还是空间换时间，取决于具体场景的资源约束。

🔑 核心结论
算法的质量决定了程序的效率。五大特性和四大设计要求是检验算法的标准。算法效率分析的核心是大O时间复杂度——通过推导大O阶，忽略常数和低阶项，只关注增长趋势。常见时间复杂度从O(1)到O(n!)共有8个层级，不同复杂度的算法在面对大规模数据时差距惊人。好的算法=用更聪明的方式让计算机少做事。学会分析时间复杂度，是成为优秀程序员的必修课。

第3章 · 线性表

本章从幼儿园小朋友排队的故事切入，生动引出线性表的定义：零个或多个数据元素的有限序列。作者系统对比了两种存储实现方式：顺序存储结构（数组）——读取O(1)但插入删除O(n)；链式存储结构（单链表）——插入删除更灵活但读取需O(n)。在此基础上，还介绍了链表的三种变体：静态链表（用数组模拟链表）、循环链表（尾结点指向头结点）、双向链表（每个结点有前驱和后继两个指针）。理解线性表是理解栈、队列等所有后续结构的前提。

线性表定义
线性表的定义：排队中的数学抽象
线性表（List）：零个或多个数据元素的有限序列。第一个元素无前驱，最后一个元素无后继，其余每个元素有且只有一个前驱和一个后继。作者用幼儿园小朋友一个拉着一个衣服排队的场景来比喻——次序固定，前驱后继关系明确。线性表的抽象数据类型（ADT）定义了基本操作：初始化、判断空、清空、获取元素、定位元素、插入、删除、获取长度等。这一定义是所有线性结构（栈、队列、串、数组等）的共同祖先。

02
顺序存储
顺序存储结构：用数组实现的线性表
用一段地址连续的存储单元依次存储线性表的数据元素——这就是顺序表。读取操作：只需O(1)时间，因为可以通过下标直接计算地址 LOC(aᵢ) = LOC(a₁) + (i-1)×c。插入操作：要在第i个位置插入元素，需要将i到n位置的所有元素向后移动一位，时间复杂度O(n)。删除操作：同理，需要将i+1到n位置的所有元素向前移动一位，时间复杂度O(n)。顺序表的优缺点非常鲜明：读取极快，插入删除极慢。
📐 地址计算公式：LOC(aᵢ) = LOC(a₁) + (i-1) × c
其中c为每个元素占用的存储单元数。这意味着存取时间性能为O(1)——这是顺序表最大的优势。
03
链式存储
链式存储结构：单链表的节点与操作
单链表用"结点"存储数据，每个结点包含数据域（data）和指针域（next），指针指向下一个结点的地址。n个结点链接成一个链表，即线性表的链式存储结构。头指针（head）指向第一个结点，最后一个结点的指针为空（NULL）。读取第i个元素：必须从头结点开始遍历i次，时间复杂度O(n)。插入和删除：如果已知位置，只需修改指针指向即可，时间复杂度O(1)；但找到位置本身需要O(n)的遍历。整表创建有两种方法：头插法（每次新结点插在头部）和尾插法（每次新结点插在尾部）。

04
存储方式对比
顺序 vs 链式：场景决定最优选择
如果应用场景以查找/读取为主，插入删除很少，选顺序存储结构。如果应用场景插入删除频繁，元素个数变化大，选链式存储结构。具体对比如下——空间：顺序需要预分配（可能浪费或不够），链式按需分配。时间：顺序读取O(1)、插入删除O(n)；链式读取O(n)、插入删除O(1)（已知位置时）。弹性：顺序表容量固定，链表可动态扩展。没有绝对的优劣，只有适合不适合——这是数据结构选型的核心思维。
📦

选顺序存储

频繁查找/读取·元素个数稳定·内存连续访问快·缓存友好

🔗

选链式存储

频繁插入/删除·元素个数变化大·不需预知容量·动态分配

05
链表变体
三种链表变体：超越单链表的更多可能
①静态链表：用数组来描述链表——数组元素由data和cur（游标）组成，cur存放下一个元素的下标。解决了没有指针的语言（如早期Basic）实现链表的问题。②循环链表：将单链表中终端结点的指针由NULL改为指向头结点，形成环。好处是从任意结点出发都能遍历全表。③双向链表：每个结点有prior（前驱指针）和next（后继指针）两个指针域。可以双向遍历，但每个结点需要多存储一个指针，空间换时间。

🔑 核心结论
线性表是最基础的数据结构。顺序存储（数组）读取O(1)但插入删除O(n)；链式存储（链表）插入删除灵活但读取需O(n)。没有绝对的优劣——频繁查找选顺序，频繁增删选链表。此外，静态链表、循环链表、双向链表分别针对不同场景提供了优化方案。理解线性表的选择逻辑，就掌握了数据结构选型的核心思维：场景决定结构。

栈的定义与实现
栈的定义与顺序存储：后进先出的精髓
栈（Stack）：限定仅在表尾进行插入和删除操作的线性表。允许操作的一端叫栈顶（top），另一端叫栈底（bottom）。LIFO（Last In First Out）后进先出结构。类比：手枪弹夹，后压入的子弹先射出。顺序栈：用数组实现，top变量指示栈顶位置。空栈时top=-1，满栈时top=MAXSIZE-1。进栈（push）：S->top++；S->data[S->top]=e。出栈（pop）：*e=S->data[S->top]；S->top--。两步操作，均为O(1)。

两栈共享与链栈
两栈共享空间与链式栈：更灵活的选择
两栈共享空间：一个数组存储两个栈，栈底分别在数组两端（下标0和n-1），两栈向中间增长。栈满条件：top1+1==top2。这就像两人合租一套两居室——共享公共空间，比各自租一居室更高效利用资源。链栈：用链表实现栈，栈顶放在链表头部。进栈=头插法，出栈=删除头结点。链栈最大优势：不需要预分空间，不会栈满溢出（除非内存耗尽）。对比顺序栈：顺序栈适合元素个数变化不大、追求速度的场景；链栈适合元素个数变化大、对空间利用率要求高的场景。
🏠 比喻：两栈共享空间
两个大学室友合租一套两居室，各住各的卧室，共享客厅厨房——比各自租一居室省钱且空间利用率更高。数组也同样：一个数组容纳两个栈，两端向中间延伸，空间利用率最大化。
03
栈的应用
栈的应用：从递归到四则运算表达式求值
递归：一个直接调用自己或通过一系列调用语句间接调用自己的函数。递归的执行过程就是栈的"压入—弹出"过程：每次函数调用将参数、局部变量等压入栈帧，返回时弹出。以斐波那契数列F(n)=F(n-1)+F(n-2)为例——虽然递归代码优雅，但因大量重复计算，效率不如迭代。四则运算表达式求值：中缀表达式→后缀表达式（逆波兰表示法），再用栈计算。后缀表达式不需要括号，操作数遇到数字压栈，遇到运算符弹出两个操作数计算后压回——这是编译器和计算器的核心原理。

队列定义与循环队列
队列的定义与循环队列：先进先出的艺术
队列（Queue）：只允许在一端进行插入操作、在另一端进行删除操作的线性表。FIFO（First In First Out）。类比：排队买票，先到的人先得到服务。顺序队列用数组实现，front指向队头，rear指向队尾的下一个位置。但顺序队列存在"假溢出"问题：rear已到数组末端但前端还有空位，无法再入队。循环队列：把数组首尾相接，形成一个环。rear=(rear+1)%MAXSIZE实现循环移动。空队列条件：front==rear；满队列条件：(rear+1)%MAXSIZE==front（保留一个空位来区分空满）。
⚠️ 假溢出问题：
顺序队列中，rear到达数组末尾后即使前面有可用空间也无法继续入队——这就是"假溢出"。循环队列通过取模运算把数组变成环形：(rear+1) % QueueSize——rear回到0后继续循环利用空间。
05
链队列
队列的链式存储：链队列的实现
链队列：用链表实现的队列，需要front指针指向队头结点，rear指针指向队尾结点。入队时在链表尾部插入结点（rear->next=new_node；rear=new_node），出队时删除头部结点（front=front->next）。空队列判断：front==rear（都指向头结点）。链队列不会有"假溢出"问题，也不需要预分空间，适合元素个数变化大的场景。对于确定最大值的情况，建议使用循环队列；无法估计长度时，使用链队列。
🔄

循环队列（顺序）

数组实现·取模循环·O(1)操作·需预知最大容量·无指针开销

🔗

链队列（链式）

链表实现·front+rear指针·O(1)操作·动态分配·无假溢出

🔑 核心结论
栈（LIFO）和队列（FIFO）是两种受限的线性表——限制操作位置非但不是缺陷，反而带来了精准的控制能力。栈的push/pop均为O(1)，两栈共享空间提升数组利用率，链栈无容量限制。栈的应用包括递归和四则运算表达式求值（后缀表达式）。队列通过循环队列解决假溢出，链队列动态分配空间。核心思想：限制不是削弱——通过精确限制操作位置，栈和队列各自成为了解决特定问题的利器。

基础概念
串的定义与比较：字符序列的数学抽象
串（string）是由零个或多个字符组成的有限序列 s="a₁a₂…aₙ"，n为串长。空串长度为零（"" 或 Φ），空格串是由空格组成的非空串。子串是串中任意连续字符组成的子序列，子串位置即其首字符在主串中的序号。串的比较基于字符编码：两个串相等⇔长度相等且每个对应位置字符均相等；不等时，按字典序逐位比较，直到发现首个不同字符或短串结束。

暴力匹配
朴素模式匹配：双重循环的代价
朴素算法对主串S每个位置i作为起点，与模式串T逐字符比较。匹配成功则返回位置，失败则i后移一位、j重置为1。最好情况：首位置即匹配→O(1)；平均情况：首字符不匹配→O(n+m)；最坏情况：每次不匹配均在T的最后一个字符才发现（如S="000…001"对T="0000000001"）→O((n-m+1)·m)。这一低效在计算机二进制01串的处理中尤为致命。

04
核心突破
KMP模式匹配：好马不吃回头草
KMP算法的核心思想：主串i值不回溯，仅变化j值。当T="abcdex"（无重复字符）时，一旦出现不匹配，可直接将j重置为1；当T="abcabx"（前缀"ab"=后缀"ab"）时，j由6退至3继续匹配。next[j]记录T串中位置j之前子串的前后缀最大相等长度：前后缀一个字符相等→k=2，n个相等→k=n+1。算法分两步：①预处理T得next数组O(m) ②模式匹配O(n)，总复杂度O(n+m)。KMP在大量"部分匹配"场景下优势显著。

精益求精
nextval改进：消除冗余回溯判断
KMP仍有缺陷：当T中多个连续字符相同时（如"aaaaax"），next回溯会产生多余比较。例如主串S="aaaabcde"匹配T="aaaaax"时，i=5,j=5不匹配后j依次退至4→3→2→1，每次比较的仍是"b"与"a"，完全多余。改进方案nextval：计算next时，若T[i]与T[next[i]]相等，则nextval[i]=nextval[next[i]]，跳过不必要的回溯。例如T="aaaaaaaab"→next=012345678→nextval=000000008，大幅减少冗余步骤。

🔑 核心结论
串是线性表的字符化扩展，核心操作聚焦于子串查找与匹配。朴素模式匹配的最坏时间复杂度O((n-m+1)·m)在二进制数据处理中堪称灾难。KMP算法通过预处理next数组实现主串不回溯，将复杂度降至O(n+m)；nextval改进针对重复字符序列进一步消除冗余回溯。回文诗《璇玑图》以841字可成7958首诗——字符串的逆转与匹配不仅是算法的艺术，更是文化的密码。

树的定义与基本概念：从一棵树到一个森林

存储方案
三种存储结构：双亲·孩子·孩子兄弟
双亲表示法：顺序存储+parent域，找双亲O(1)，找孩子需遍历。可扩展长子域、右兄弟域来优化。孩子表示法：顺序存储表头结点+每个结点的孩子单链表，兼顾遍历与寻子。孩子兄弟表示法（二叉链表法）：每个结点设firstchild和rightsib两个指针——这是将任意树转换为二叉树的关键桥梁。存储结构设计没有银弹，需根据运算场景灵活选择。

遍历体系
二叉树的四序遍历：把树变成线性序列
二叉树每个结点最多两棵子树，左右有序。五种基本形态：空树、仅根、仅左子、仅右子、左右兼有。特殊二叉树：左斜树/右斜树、满二叉树（所有层全满）、完全二叉树（按层序编号无空档）。五大性质：第i层至多2ⁱ⁻¹结点、深度k至多2ᵏ-1结点、n₀=n₂+1、深度⌊log₂n⌋+1、父子下标关系2i与2i+1。四种遍历将树转化为线性序列：前序（根→左→右）、中序（左→根→右）、后序（左→右→根）、层序（逐层从左到右）。已知前序+中序或后序+中序可唯一确定二叉树；仅知前序+后序不可。

空间优化
线索二叉树：把空指针变成GPS导航
n个结点的二叉链表有2n个指针域，但仅n-1条分支，存在n+1个空指针域。线索化的原理：利用这些空指针存储结点在某种遍历次序下的前驱和后继——就像GPS导航仪告诉你下一步该往哪走。增设ltag/rtag标志区分「指向孩子」(0)与「指向线索」(1)。线索二叉树等价于双向链表，遍历时间复杂度O(n)，同时省去每次遍历找前驱后继的开销。通过添加头结点，可从第一个结点顺后继遍历到尾，或从最后一个结点顺前驱遍历到头。

压缩基石
赫夫曼树与编码：数据压缩的数学之美
赫夫曼树是带权路径长度（WPL）最小的二叉树，构建算法：每次取两个最小权值结点合并为新结点，权值相加。赫夫曼编码：以字符频率为权构造赫夫曼树，左分支为0，右分支为1，从根到叶的路径即为该字符的编码。核心特性：任一字符编码都不是另一个的前缀（前缀编码），确保解码无歧义。例如字母频率A27/B8/C15/D15/E30/F5，赫夫曼编码将30位二进制串压缩为25位，节省约17%空间。这是无损压缩的理论基石，ZIP、JPEG等现代压缩技术皆源于此。

🔑 核心结论
