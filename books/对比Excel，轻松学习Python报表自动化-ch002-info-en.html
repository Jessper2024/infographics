Compare with Excel, Easily Learn Python Report Automation · Ch.2 "Python Fundamentals"

SCQA · Situation → Complication → Question → Answer Python, with its simple syntax and rich library ecosystem, is the implementation tool for report automation. But Excel users with no programming background face a steep learning curve: environment setup, Jupyter Notebook, variables, data types, data structures, loops, functions. How to master the essential Python knowledge for report automation in the shortest time? This chapter focuses on exactly what's needed—from Anaconda installation to functions and libraries—with code examples for every concept, just enough to get started, not exhaustive.

This chapter is a crash course for Excel users with zero Python experience. It starts with Anaconda installation (tutorials for both Windows and Mac), then introduces Jupyter Notebook—the dominant IDE in data analysis. It systematically covers Python core syntax: number types, variables and identifiers, print output and formatting, indentation and comments. It delves into nine string operations (concatenation, replication, length, search, index, split, strip), and three core data structures—list, dict, and tuple—with their creation and manipulation. Finally it covers operators, loops (for/while), conditionals (if/elif/else), functions (def and lambda), and advanced features like list comprehensions and map().

02
Basics
Variables, Data Types, Output & Indentation
Numbers are int or float (judged by decimal point). Variables are containers for data, named starting with a letter or underscore. print() outputs data; str.format() enables flexible formatting (.2f for decimals, .2% for percentages). Python uses indentation (usually 4 spaces) to identify code blocks; # for single-line comments.

Variable = Container

Stores any data type. Naming: letter/underscore start, case-sensitive, not a keyword.

03
Data Handling
String Operations & Three Core Data Structures
Strings support nine operations: + concatenation, * replication, len(), in/find(), [] indexing, [:] slicing, split(), strip(). Lists are mutable ordered collections with append/insert/pop/remove/sort. Dicts are key-value pairs with keys()/values()/items(). Tuples are immutable sequences; list() and tuple() enable mutual conversion.

04
Control Flow
Loops, Conditionals & Functions
for loops iterate over each element in a sequence. while loops continue until the condition becomes false (beware infinite loops). if/elif/else chains form conditional logic. Functions are defined with def, can take parameters and return values using return. Anonymous lambda functions offer concise one-liners: lambda arg: expression.
# for loop — iterate through courses for course in ["Excel","SQL","Python","Stats"]: print(f"Learn {course}") # lambda — anonymous function multiply = lambda x, y: x * y print(multiply(3, 4)) # Output: 12 # if/elif/else — conditional chain if score >= 80: print("Excellent") elif score >= 60: print("Pass") else: print("Fail")
05
Advanced
List Comprehensions, map() & Library Management
List comprehensions replace multi-layer for loops in one line: [x**2 for x in list]. map(function, args) applies a function to each element; use list() to convert results. Libraries are cross-program callable function collections: pip install xxx to add, pip uninstall xxx to remove, import to use. This book primarily uses openpyxl.

🔑 Key Takeaway
Python's syntax isn't complex—it was designed as "executable pseudocode." The six modules covered in this chapter—variables, data structures, loops, conditionals, functions, and libraries—constitute all the programming foundation needed for report automation. Master these, and the rest of the book is essentially learning the openpyxl API.
