Excel vs Python Report Automation · Ch7「Setting Excel Cells with Python」

SCQA · Situation → Complication → Question → Answer Daily Excel worksheet maintenance involves inserting rows, deleting rows, adjusting row heights/column widths, and hiding rows/columns. Manually, it's right-click → Insert → right-click → Delete → right-click → Row Height → right-click → Column Width—repeated endlessly. When you have dozens of sheets all needing the same adjustments, manual work is not only time-consuming but also prone to misfires. How do you automate Excel's cell row/column structure management with Python? This chapter covers insert_rows/insert_cols, delete_rows/delete_cols, row/column dimension settings, row/column hiding (via group), and a batch setup case study.

This chapter focuses on the Python implementation of Excel's Cells group on the Home tab, covering four operations: ① Insert rows/columns—insert_rows(m, n) and insert_cols(m, n), where m is the position and n is the count (default 1); ② Delete rows/columns—delete_rows(m, n) and delete_cols(m, n), the inverse of insertion with identical parameters; ③ Row height/column width—set via row_dimensions[row_num].height and column_dimensions[col_letter].width; ④ Row/column hiding—achieved through group combinations (column_dimensions.group and row_dimensions.group) since openpyxl has no direct hide API, with the group approach being more flexible as it supports collapse/expand. The chapter concludes with a batch case study showing how to iterate rows and columns for uniform height/width settings.

Insert
insert_rows / insert_cols: Dynamically Expand Worksheet Structure
insert_rows(m, n) inserts n rows before row m; insert_cols(m, n) inserts n columns before column m. n defaults to 1 and can be omitted. This is the Python equivalent of Excel's right-click Insert command. Existing data automatically shifts. Ideal for adding data regions, reserving blank rows, or inserting separator rows.

02
Delete
delete_rows / delete_cols: Precisely Remove Excess Rows/Columns
delete_rows(m, n) deletes n rows starting from row m; delete_cols(m, n) deletes n columns starting from column m. n defaults to 1. This is the inverse of insertion, equivalent to Excel's right-click Delete command. Used to clean empty rows, remove unwanted data columns, and slim down report structure. Note: deletion is irreversible—back up the original sheet first.

03
Dimensions
Row Height & Column Width: row_dimensions / column_dimensions
Row height: ws.row_dimensions[row_num].height; column width: ws.column_dimensions[col_letter].width. This matches Excel's right-click Row Height/Column Width dialog. Change the row number or column letter to adjust other rows/columns—simple and intuitive.
# Set row 1 height to 40, column A width to 20 ws.row_dimensions[1].height = 40 ws.column_dimensions['A'].width = 20 # Batch: iterate row numbers / column letters for r in [1, 2, 3]: ws.row_dimensions[r].height = 40 for c in ['A', 'B', 'C']: ws.column_dimensions[c].width = 20
04
Hide
Row/Column Hiding: Collapsible Regions via Group
In openpyxl, rows/columns are hidden by creating groups rather than a direct hide API. ws.row_dimensions.group(7, 10) hides rows 7-10; ws.column_dimensions.group('D', 'F') hides columns D-F. This approach is more flexible than simple hiding—it supports collapse/expand, allowing users to manually reveal hidden data in Excel.

Batch Row/Column Dimension Setup: Standardize Entire Table
Insert/delete operations are typically one-off actions, but row/column dimensions often need table-wide uniform settings. By iterating row numbers or column letters in a for loop, setting height/width for each row/column, you can standardize the entire table's dimensions in one pass. Write once, reuse forever.

🔑 Key Takeaway
Excel's row/column structure management—insert, delete, row height, column width, hide—maps to four pairs of functions/properties in Python: insert_rows/insert_cols, delete_rows/delete_cols, row_dimensions.height/column_dimensions.width, and group-based hiding. These APIs directly mirror Excel's right-click menu functions with a flat learning curve. In practice, insert/delete are typically one-off operations while row/column dimensions require batch iteration across the table. One for loop across all target rows and columns, uniformly setting height and width, standardizes the entire worksheet layout in one go. Python turns Excel's "manual labor" into "intellectual work".
