List Definition
Definition of Linear List: Mathematical Abstraction of Queueing
Linear List: a finite sequence of zero or more data elements. The first element has no predecessor, the last has no successor, and every other element has exactly one predecessor and one successor. The author uses the analogy of kindergarten children holding each other's clothes in line — fixed order, clear predecessor-successor relationships. The List ADT defines basic operations: init, isEmpty, clear, getElement, locateElement, insert, delete, getLength. This definition is the common ancestor of all linear structures (stacks, queues, strings, arrays).

02
Sequential Storage
Sequential Storage: Implementing Lists with Arrays
Using a contiguous block of memory addresses to store list elements sequentially — this is the sequential list. Read operation: only O(1) time, because the address can be directly computed: LOC(aᵢ) = LOC(a₁) + (i-1)×c. Insertion: to insert at position i, all elements from i to n must shift backward — O(n). Deletion: similarly, all elements from i+1 to n must shift forward — O(n). Sequential lists have very clear pros and cons: blazing fast reads, painfully slow insertions/deletions.
📐 Address Formula: LOC(aᵢ) = LOC(a₁) + (i-1) × c
where c is the storage size per element. This means O(1) access time — the biggest advantage of sequential lists.
03
Linked Storage
Linked Storage: Nodes and Operations of Singly Linked Lists
A singly linked list stores data in "nodes," each containing a data field and a pointer field (next) that points to the next node's address. n nodes linked together form a linked list — the linked storage structure of a linear list. The head pointer points to the first node; the last node's pointer is NULL. Reading element i: must traverse from the head, i iterations — O(n). Insertion/Deletion: if the position is known, only pointer modifications are needed — O(1); but finding the position itself requires O(n) traversal. List creation has two methods: head insertion (each new node at the front) and tail insertion (each new node at the back).

04
Storage Comparison
Sequential vs Linked: The Scenario Decides the Winner
If your scenario is search/read-heavy with few insertions/deletions, choose sequential storage. If insertions/deletions are frequent with varying element counts, choose linked storage. Specific comparison — Space: sequential requires pre-allocation (may waste or be insufficient); linked allocates on demand. Time: sequential reads O(1), insert/delete O(n); linked reads O(n), insert/delete O(1) (when position is known). Flexibility: sequential capacity is fixed; linked can dynamically expand. There is no absolute winner — only fitness for purpose. This is the core mindset of data structure selection.

05
List Variants
Three List Variants: Beyond the Singly Linked List
① Static Linked List: describes a linked list using an array — array elements consist of data and cur (cursor), where cur stores the index of the next element. Solves the problem of implementing linked lists in languages without pointers (e.g., early BASIC). ② Circular Linked List: changes the terminal node's pointer from NULL to point to the head node, forming a ring. Advantage: can traverse the entire list from any node. ③ Doubly Linked List: each node has prior (predecessor pointer) and next (successor pointer). Enables bidirectional traversal, but each node requires one extra pointer — trading space for time.

🔑 Key Takeaway
The linear list is the most fundamental data structure. Sequential storage (array) offers O(1) reads but O(n) insertions/deletions; linked storage (list) offers flexible insertions/deletions but O(n) reads. There is no absolute winner — choose sequential for frequent searches, linked for frequent modifications. Additionally, static, circular, and doubly linked lists provide optimizations for different scenarios. Understanding the selection logic of linear lists means mastering the core mindset of data structure design: the scenario determines the structure.
