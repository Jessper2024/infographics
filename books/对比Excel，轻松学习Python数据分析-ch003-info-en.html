📦 Chapter Overview
What Python structures hold spreadsheet-like data? What exactly are Series and DataFrame?
This chapter marks the starting point of the Practice section, officially entering the data analysis process. It focuses on Pandas' two core data structures—Series (one-dimensional indexed array) and DataFrame (two-dimensional tabular structure with row and column indices). A Series corresponds to a single column in Excel, while a DataFrame corresponds to an entire worksheet. By comparing with Excel's data presentation, this chapter builds the cognitive mapping from spreadsheets to Python data structures, laying the groundwork for all subsequent data operations. It also establishes the pd/np/plt module alias conventions.

Mental Shift: Excel to Pandas

Chapter 3 marks the turning point from "learning tools" to "using tools." From here onward, every chapter revolves around a specific data analysis operation, contrasting Excel and Python implementations.

  • Three Module Conventions: Pandas → pd, NumPy → np, matplotlib.pyplot → plt. Import once per program; aliases used throughout the book.
  • Excel → Pandas Mindset: One Excel column ≈ one Series; one Excel worksheet ≈ one DataFrame. Row index + column index determine a unique value, mapping directly to Excel's row-number/column-name concept.
  • Key Difference: Excel counts headers in row/column totals; Python's shape method excludes indices.
2
Series: One-Dimensional Labeled Array

A Series consists of one set of data (values) and one set of associated labels (index), analogous to a single column with row numbers.

  • Creating a Series — pd.Series():
    ① Pass a list → default index starting from 0
    ② Pass a list + index parameter → custom data labels
    ③ Pass a dictionary → keys become data labels, values become data values
    ④ Can also pass a constant + index parameter
  • Get Index: series.index
  • Get Values: series.values
  • Excel Analogy: A Series is like a single column in Excel (without the column name); the left-side indices correspond to row numbers.
3
DataFrame: Two-Dimensional Tabular Structure

DataFrame consists of data paired with two indices (row index and column index)—the most important tabular data structure. It's called "tabular" precisely because its form closely resembles Excel's data storage format.

  • Creating a DataFrame — pd.DataFrame():
    ① Single list → one column, default row/column indices
    ② Nested list → multiple columns (each nested list = one column)
    ③ Nested list + columns parameter → custom column indices
    ④ Nested list + index parameter → custom row indices
    ⑤ Dictionary → keys as column indices, values as column data
  • Excel Analogy: DataFrame ≈ a complete Excel worksheet. Row index = row numbers; column index = column headers.
4
Accessing DataFrame Indices and Values

Mastering DataFrame index access is the prerequisite for all subsequent data selection operations:

  • Get Column Index: df.columns — returns all column names. Like viewing Excel's header row.
  • Get Row Index: df.index — returns all row labels. Like viewing Excel's row numbers.
  • Get Values: df.values — returns DataFrame data as a plain array (excluding indices). This is the foundation for Chapter 6 "Data Selection."

Understanding row and column indices underpins every subsequent data operation—whether it's data selection (Ch.6), value replacement (Ch.7), arithmetic operations (Ch.8), or multi-table joins (Ch.11), the fundamental mechanism is always locating and manipulating data within the row/column coordinate system.

5
Module Imports & Book-Wide Coding Conventions

Importing the three core modules is the universal prerequisite for all Python code in this book:

  • import pandas as pd — Pandas is the central module; used in virtually every chapter
  • import numpy as np — NumPy provides numerical computing foundations; Pandas is built on NumPy
  • import matplotlib.pyplot as plt — matplotlib handles data visualization (Chapter 13)

In code examples, the "pd" in pd.Series(), pd.DataFrame(), pd.read_excel(), etc., is exactly the Pandas alias. Using as for aliasing is a universal Python community convention—adopted here for brevity and readability.

💡 Key Takeaway

Series and DataFrame are the two cornerstones of Pandas. If you understand Excel's row/column concept, Pandas data structures are easy to grasp—Series = one data column + row index, DataFrame = multiple data columns + row index + column index. Though short, this chapter provides the "grammar foundation" for all subsequent operations (importing, cleaning, selecting, computing, grouping, joining, visualizing). Remember: with pd/np/plt imported once, all the book's code becomes reusable—that's the power of Python's modularity.
