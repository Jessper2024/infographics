Data Filtering · Precise Targeting of Data

This chapter focuses on precise data selection and filtering, using "picking ingredients" as a metaphor. It systematically covers column selection, row selection, and combined row-and-column selection. Through Excel-Python comparisons, it details four indexing methods—nominal (loc), positional (iloc), boolean, and slice indexing—and their combinations, covering all data extraction scenarios from single-column selection to conditional cross-section filtering.

1
Column Selection

The first dimension of data selection, covering single column, multiple columns, and consecutive columns:

  • Nominal indexing: df['col_name'] for single, df[['col1','col2']] for multiple
  • Positional indexing: df.iloc[:, [0,2]] selects by column position (0-indexed)
  • Slice indexing: df.iloc[:, 0:3] selects consecutive columns, left-inclusive right-exclusive

In Excel, simply mouse-select or Ctrl+click. In Python: use names for nominal, use iloc for positional/slice.

2
Row Selection

Select specified row data, supporting single row, multiple rows, consecutive rows, and conditional filtering:

  • Nominal (loc): df.loc[['A1','A2']] select by row index name
  • Positional (iloc): df.iloc[[0,2]] select by row position
  • Consecutive rows: df.iloc[0:3] slice to get consecutive rows
  • Conditional (boolean): df[df['age']<200] filter rows meeting condition; supports multi-condition (&, |)

Excel row selection relies on filter functionality. Python's boolean indexing is the most flexible row selection method.

3
Combined Row-Column Selection

Select intersection of rows and columns simultaneously, with five combination patterns:

  • Nominal+Nominal (loc): df.loc[['row_names'], ['col_names']]
  • Positional+Positional (iloc): df.iloc[[row_pos], [col_pos]]
  • Boolean+Nominal: df[df['condition']][['col_names']] — filter rows then select columns
  • Slice+Slice (iloc): df.iloc[0:2, 1:3] — both rows and columns by interval
  • Cross indexing (ix): mix nominal and slice indexing (gradually deprecated)

Core principle: use loc for names, iloc for positions — the indexing method determines the accessor.

4
Index Method Summary

Python's four core indexing mechanisms for data selection:

  • Nominal indexing: pass row/column names, use loc method — intuitive for known-name scenarios
  • Positional indexing: pass row/column position numbers (0-based), use iloc method — suitable for batch processing
  • Boolean indexing: pass logical conditions, returns matching rows — most flexible, supports complex filtering
  • Slice indexing: pass position intervals (start:end), use iloc method — suitable for contiguous region selection

These four methods form Python's data selection foundation; any combination covers all data extraction scenarios.

5
Tool Comparison Highlights

Key differences between Excel and Python for data selection:

  • Excel relies on mouse operations + filter functionality, suitable for ad-hoc data exploration
  • Python uses code-based selection, suitable for automated, reproducible data processing workflows
  • Python's boolean indexing enables multi-condition combined filtering, far exceeding Excel's filter flexibility
  • Combined row-column selection has five patterns in Python, while Excel can only do it by mouse drag
📊 Chapter Overview

Data selection is the key skill of "precise targeting." By mastering four methods—loc (by name), iloc (by position), boolean indexing (by condition), and slice indexing (by range)—and their combinations, you can quickly extract target data in any analytical scenario. Excel mouse operations suit exploration, while Python code-based selection suits automated production; together they form a complete data selection capability.
