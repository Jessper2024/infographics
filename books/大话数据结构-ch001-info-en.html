Origin Story
Xiaocai's Lesson: Why Data Structures Matter
Xiaocai built a customer queue system using a database table with auto-increment IDs. The system crashed under load — he had no idea that a "queue" data structure existed. After working overtime to rewrite it with an array, he was told arrays could overflow. Finally with his cousin's help, he rewrote it properly using a queue structure and passed the review. Without knowledge of data structures, you will solve already-solved problems the wrong way. Data structures study the objects, relationships, and operations involved in non-numerical programming problems.

02
Concept Hierarchy
From Data to Data Structure: A Five-Layer Tower
Data: symbols describing objective things that can be input into and processed by computers. Data Element: the basic unit composing data, usually processed as a whole. Data Item: the smallest indivisible unit of a data element. Data Object: a collection of data elements sharing the same properties, a subset of data. Data Structure: a collection of data elements with one or more specific relationships among them. This five-layer concept system is the foundation for all subsequent chapters.
① Data
Outermost Layer

② Data Object

03
Logical Structure
Four Logical Structures: How Data Elements Relate
Logical structure describes relationships among data elements, divided into four types: Set (elements belong to the same set, no other relation), Linear (one-to-one), Tree (one-to-many hierarchical), and Graph (many-to-many arbitrary). Logical structures are problem-oriented abstractions. For example: children lining up form a linear structure, a corporate org chart forms a tree structure, and a city traffic network forms a graph structure. These four structures cover virtually all real-world modeling needs.

04
Physical Storage
Two Physical Structures: Sequential and Linked
Physical (storage) structure is how data structures are actually stored in computer memory. Sequential storage: elements are placed in contiguous memory addresses, with logical relationships matching physical ones — like people standing right next to each other in a line. Linked storage: elements are placed in arbitrary memory locations, connected through pointers — like patients holding number tickets in a hospital waiting room. Sequential excels at reads; linked excels at insertions and deletions.

05
Abstract Data Type
ADT: From "What It Is" to "What It Can Do"
Abstract Data Type (ADT): a mathematical model and a set of operations defined on that model. ADT does not care about implementation details — only "what it is" and "what it can do." For example: the Integer ADT includes add, subtract, multiply, divide without caring how they are implemented. The List ADT includes init, find, insert, and delete operations. ADT bridges "data structure concepts" and "concrete programming implementation" — first define what data can do, then implement how. This embodies the core of object-oriented thinking: encapsulation and abstraction.
🧠 Key Insight: ADT = Data Object + Data Relationships + Basic Operations. "Abstract" means hiding implementation complexity, exposing only an operational interface. "Data Type" means it works like int or float — providing a set of defined operations.
🔑 Key Takeaway
A data structure is a collection of data elements with one or more specific relationships among them. Four logical structures (Set, Linear, Tree, Graph) plus two physical storage methods (Sequential, Linked) form the foundation of all data structure knowledge. The Abstract Data Type (ADT) encapsulates data definitions with their operations, serving as the cornerstone of object-oriented thinking. Learning data structures is not for exams — it is about solving already-solved problems the right way at work.
